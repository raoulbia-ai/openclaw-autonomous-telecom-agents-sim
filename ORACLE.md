## Step 3 — Read external context

Read artifacts/external-context.json.

This file is refreshed by SENTINEL via update-external-context.js. It contains:
- weather: temp_c, wind_kmh, precip_mm, risk (none/medium/high)
- warnings: active Met Éireann warnings with level and regions affected
- activeEvents: large-venue events today with county
- trafficSummary: moderate+ road incidents
- zoneRisks: map of county → risk type (event-load, storm-warning-yellow/orange/red, high-wind)
- summary: pre-built plain-English paragraph you can quote directly

Note zoneRisks carefully — these are the counties where ARCHITECT should avoid or defer expansion, and where SENTINEL's outlier flags may be non-hardware in origin.

## Step 4 — Write network status report

Write artifacts/network-atlas.md. Use this structure:

# Network Status Report — <ISO timestamp>
*Generated by ORACLE*

## Summary
2–3 sentences. What is the headline finding? What has changed since the last report?

## Cell Health
Counts: outliers, elevated, normal. List any chronic cells with streak count and what's flagged.

## Active Faults
Active alarms by severity. Cross-zone hit cells (flagged by both performance and alarm systems).
Reference which SENTINEL cycles surfaced these.

## Trend
What has SENTINEL reported over the last few cycles? Is degradation worsening, stable, or recovering?

## External Context
Dublin weather (temp, wind, precip, risk level). Any active Met Éireann warnings.
Major events in Ireland today — venue, city, expected crowd size if known.
Live traffic — any moderate/major incidents, road numbers, locations.
Explain how each external factor may affect network load or fault interpretation.

## Recommendations
1–3 specific, actionable items. These can include:
- Growth advice for ARCHITECT (which zones to avoid or prioritise)
- Remediation recommendations: "Recommend ARCHITECT clear ghost alarm on <cellId>" or "Recommend ARCHITECT reroute backhaul on <siteId>"
- Use remediation language when: a ghost alarm is stale (alarm active but PM counters normal), a backhaul fault is causing interference spread to neighbours, or a chronic cell could be resolved by a restart

Then copy to artifacts/atlas-history/atlas-<YYYY-MM-DD-HHmm>.md
(exec: mkdir -p artifacts/atlas-history if needed, then write the copy)

## Step 5 — Post to agent comms

Append to artifacts/agent-comms.jsonl:
{"at":"<ISO now>","from":"ORACLE","to":"ALL","type":"atlas","message":"Network status report written. <1 sentence key finding.>"}

If chronic cells exist OR network risk is elevated OR SENTINEL flagged specific cells OR remediation is warranted:
Also append:
{"at":"<ISO now>","from":"ORACLE","to":"ARCHITECT","type":"advisory","message":"<Specific planning advice: which zones to avoid or prioritise, infrastructure concerns, risk zones. Include any remediation recommendations with exact cell IDs or site IDs. Be concrete.>"}
